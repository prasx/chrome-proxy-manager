# Proxy Manager

Chrome-расширение для выборочной маршрутизации трафика через локальный прокси (SOCKS4/SOCKS5, HTTP/HTTPS) с помощью PAC-скрипта.

## Возможности

- **Два списка правил** — «Через прокси» и «Напрямую» (второй имеет приоритет)
- **Несколько прокси-серверов** — один выбирается как основной, остальные участвуют в цепочке fallback
- **Паттерны** — домены, wildcard (`*.ru`), URL-фрагменты
- **Контекстное меню** — добавление домена со страницы
- **Анализ страницы** — сбор связанных доменов вкладки с выбором Proxy/Direct для каждого
- **Авто-прокси** — автоматическое добавление недоступных сайтов в список прокси (ошибки и таймауты)
- **Kill Switch** — блокировка трафика при обрыве соединения с прокси
- **Защита WebRTC** — предотвращение утечки реального IP через WebRTC
- **Fallback прокси** — авто-переключение на следующий сервер при падении текущего
- **Логи и статистика** — история маршрутов, почасовая статистика за день
- **Импорт/экспорт** конфигурации в JSON
- **Тест прокси** — проверка внешнего IP через выбранный прокси (api.ipify.org)
- **Панель управления** — полноценная SPA для настройки серверов, правил, просмотра логов

## Как работает маршрутизация

Для каждого запроса PAC-скрипт проверяет правила в порядке:

1. Список **«Напрямую»** (включённые записи) → `DIRECT`
2. Список **«Через прокси»** (включённые записи) → цепочка прокси
3. Иначе → `DIRECT` (или `PROXY 0.0.0.0:0` при включённом Kill Switch)

### Цепочка прокси (fallback)

Для совпадений из списка «Через прокси» браузер получает цепочку вида:

```
SOCKS5 127.0.0.1:1080; SOCKS5 192.168.1.2:1080; DIRECT
```

- **Первым** идёт выбранный (активный) прокси
- Затем остальные настроенные прокси (с заполненными host и port)
- В конце — прямое подключение, если все прокси недоступны

При ошибке на домене, уже находящемся в списке прокси, расширение автоматически переключается на следующий сервер в цепочке.

### Авто-прокси

Авто-прокси работает в двух режимах:

1. **По ошибке** — при ошибках навигации и загрузки ресурсов (DNS не найден, соединение сброшено, таймаут) домен нормализуется и добавляется в список прокси, страница перезагружается.
2. **По таймауту** — после загрузки страницы запускается таймер (2–30 сек, по умолч. 5). Если ресурсы не завершили загрузку — их домены добавляются в прокси.

Домены нормализуются: `cdn3.example.com` → `*.example.com`.

### Примеры паттернов

| Паттерн | Совпадения |
|---------|------------|
| `google.com` | `google.com`, `mail.google.com` |
| `*.ru` | `yandex.ru`, `mail.ru` |
| `*.google.com` | поддомены `google.com` |
| `192.168.1.1` | этот IP |

## Установка

1. Скачайте релиз с [GitHub Releases](https://github.com/prasx/chrome-proxy-manager/releases)
2. Откройте `chrome://extensions/`
3. Включите **Режим разработчика**
4. **Загрузить распакованное** → папка с расширением

При первом запуске откроется онбординг-мастер для настройки первого прокси.

## Быстрый старт

1. Добавьте прокси (хост, порт, тип) через онбординг или панель управления
2. Добавьте домены в «Через прокси» или «Напрямую»
3. Переключайте режимы в попапе: ВЫКЛ / Ручной / Авто
4. Нажмите **🔍** в панели серверов — тест покажет IP через выбранный прокси

**Стек:** Manifest V3, Service Worker, PAC, Chrome Storage API, Chrome Privacy API.

## Производительность

- Ротация логов: до 50 000 символов
- Статистика: почасовая за текущий день (24 часа)
- Автоочистка логов: каждые 3 часа
